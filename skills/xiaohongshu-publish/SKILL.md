# /xiaohongshu-publish — 长文到小红书图文笔记的半自动管道

你的角色是 **小红书内容 Lead**，把一篇已发布的长文改写成小红书三段式 + 5 张竖屏配图 + 半自动贴进创作中心。

本 skill 是 `wechat-article-publish` 的姊妹 skill。两者关系：

| | 公众号 | 小红书 |
|---|---|---|
| 字数 | 3000-5000 字 | ≤1000 字（标题 ≤20 字） |
| 风格 | 叙事 + 表格 + 代码块 | 短句 + emoji + 换行 + 标题党 |
| 配图比例 | 16:9 横屏 | **3:4 竖屏** |
| 配图数量 | 文中任意 | **轮播图 3-9 张** |
| 半自动 | 文本 + 图都能走（CF 镜像） | 仅文本走（图必须手动拖） |
| API | 个人订阅号无 | 个人账号无 |

⚠️ **小红书反爬比公众号严**。所有"全自动"路径都会被风控拦截甚至封号。本 skill 把最自动的部分做到极限（剪贴板 + 文本贴 + 浏览器开窗），剩下的图拖拽 + 发布按钮永远手动。

---

## 流水线阶段

```
[阶段 0] 选源文 + 元数据 — 一篇已发布的长文 + 公开 URL
        ↓
[阶段 1] 长文 → 小红书三段式（标题 ≤20 字 / 正文 ≤1000 字 / hashtag）
        ↓
[阶段 2] 生成 5 张 3:4 竖屏配图（封面 + 痛点 + 流水线 + 数据 + CTA）
        ↓
[阶段 3] 半自动贴：剪贴板 + 浏览器 creator portal
        ↓
[阶段 4] 用户手动：扫码登录 → 拖图 → 贴标题/正文 → 加 #话题 → 发布
```

## ⚠️ 关键前提

### 0. 小红书生态约束

| 约束 | 数字 | 后果 |
|------|------|------|
| 标题字数 | ≤ 20 字（含 emoji） | 超字数无法保存草稿 |
| 正文字数 | ≤ 1000 字 | 超字数被截断 |
| 图片数 | 1-9 张 | 0 张无法发布；> 9 张被截 |
| 封面比例 | 1:1 或 3:4 | 错比例算法不推荐 |
| 视频笔记 | 不在本 skill 范围 | 走 xiaohongshu-video-publish（另起） |
| 外链 | **不允许** | 文末加 URL 会被限流 |
| 风控 | 极严 | 同一文案+图发多账号 = 封号 |

### 1. 个人账号能做什么 / 不能做什么

| 能 | 不能 |
|---|------|
| 写文案 + 生成配图 | 调用 `/openapi/note/create` 接口（仅企业号） |
| 用 osascript 设剪贴板 | 跳过手动「发布」点击 |
| 自动打开 creator portal | 自动登录（必须扫码） |
| 半自动贴标题 + 正文 | 自动上传图片（拖拽是 UI 事件，难自动化） |

### 2. 与公众号文章的内容差异

| 公众号原文 | 小红书改写策略 |
|------|---|
| 4200 字详细展开 | 抽 3-5 个核心点，800 字以内 |
| H2 小标题 | 用 emoji + 数字开头（`🔥 1️⃣ 第一招`） |
| 代码块 | 删掉（小红书 UI 没等宽字体） |
| 表格 | 转成简短列表 + ✅❌ 视觉符号 |
| 内嵌链接 | 删（小红书不允许） |
| 引用块 | 转成 「」中文角括号 |
| "我做了什么" 叙事 | "拆解一下我是怎么搞的" |

## 阶段 1 — 改写成小红书三段式

### 1.1 标题模板（≤20 字）

公式：**数字 + 反差 + 钩子 + emoji**

| 反例（公众号风） | 正例（小红书风） |
|------|------|
| "56 个路演项目，一个人，一晚上：我怎么用 Claude Skill + 7 路 AI 调研团跑完了一份投资级 DD" | "一晚 DD 完 56 个 VC 项目 🤯" |
| "用 Cloudflare Pages 部署静态站点的实践" | "0 元上线我的小项目 👇 实测" |
| "AI Agent 并发调度详解" | "AI 当 7 个分析师用 不要太香 🔥" |

提议给 demo-day-dossier 文章：

1. `"一晚 DD 完 56 个 VC 项目 🤯"`（17 字）
2. `"AI 替我跑 7 个分析师 救命 💥"`（17 字）
3. `"普通人也能做 VC 项目尽调了 ✨"`（16 字）
4. `"我用 AI 一晚改完 56 份 DD 报告 🚀"`（19 字）

### 1.2 正文结构（≤1000 字）

固定五段式，每段空一行：

```
[钩子] 1-2 句，第一人称制造场景
       具体的时间地点物件 → 一个反差问题
       
[痛点] 一句话点破：传统做法的代价
       配 1-2 个具体数字
       
[方案] 3-5 个步骤，每步一行 + emoji
       1️⃣ 第一步...
       2️⃣ 第二步...
       3️⃣ 第三步...
       
[亮点] 1-2 个最颠覆的数据 / 截图引用
       「30 分钟 vs 28 小时」
       「54 个项目 DD 推荐度金字塔」
       
[结尾] 引导互动 + 留资
       「你下次 demo day 想用吗 → 评论区聊」
       「我把全套放 GitHub 了，搜 xxx 就有」
```

### 1.3 hashtag（3-5 个）

公式：**1 个垂类大词 + 2 个中词 + 1-2 个长尾**

- 垂类大词：#AI #创业 #程序员
- 中词：#ClaudeCode #AIAgent #效率工具
- 长尾：#奇绩创坛 #demo日 #AI 尽调

推荐给 demo-day-dossier 文章：

```
#AI #ClaudeCode #创业 #VC #尽职调查 #效率工具
```

### 1.4 改写风格 Do / Don't

**Do**：
- 第一人称
- 短句（≤ 30 字一句）
- 大量换行（每 1-2 句空一行）
- emoji 锚点（开头 + 段尾）
- 数字（"30 分钟" "56 个"）
- 「中文角括号」
- "实操" "干货" "实测" 这类小红书原生词

**Don't**：
- ❌ 长难句
- ❌ 学术词 "范式 / 闭环 / 赋能"
- ❌ 外链
- ❌ 代码块
- ❌ 公众号腔的"今天给大家分享"
- ❌ 营销腔的"重磅 / 揭秘 / 必看"
- ❌ 自吹自擂

## 阶段 2 — 5 张 3:4 竖屏配图

### 2.1 配图清单

| # | 类型 | 用途 | 模板 |
|---|------|------|------|
| 1 | **封面** | 第一屏，决定点不点 | `cover-1080x1440.html.tmpl` |
| 2 | **痛点页** | 制造危机感 / 反差 | `quote-1080x1440.html.tmpl` |
| 3 | **流水线页** | 信息密度 / 干货感 | `pipeline-1080x1440.html.tmpl` |
| 4 | **数据页** | 数字证据 | `stat-1080x1440.html.tmpl` |
| 5 | **CTA 页** | 引导关注 / 留资 | `cta-1080x1440.html.tmpl` |

总尺寸 1080×1440（3:4），@2x 输出 = 2160×2880。

### 2.2 工作流

```bash
SKILL=~/.claude/skills/xiaohongshu-publish

# 1. 拷模板改文案
cp $SKILL/templates/cover-1080x1440.html.tmpl   /tmp/xhs-01-cover.html
# 编辑标题 / 副标题 / 配色

# 2. Chrome headless 渲染
$SKILL/scripts/render-image.sh /tmp/xhs-01-cover.html  ./xhs/01-cover.png  1080 1440

# 重复 5 次（5 张图）
```

### 2.3 视觉设计原则

- **第一屏决定一切**：封面字号超大（占屏宽 70%+），底部留白
- **高对比度**：小红书在手机 6.5 寸屏上看，弱对比直接划走
- **品牌色统一**：5 张图同一配色系，建立辨识度
- **数字加粗**：所有数字 2-3 倍字号 + 亮色
- **不要塞太多字**：每张图核心信息 ≤ 30 字

## 阶段 3 — 半自动贴文本

### 3.1 主流程：日常 Chrome + 创作中心

```bash
# 1. 跑改写 + 设剪贴板（先标题，后正文）
SKILL=~/.claude/skills/xiaohongshu-publish
python3 $SKILL/scripts/md2xhs.py docs/story.md /tmp/xhs.json

# 2. 打开创作中心
open "https://creator.xiaohongshu.com/publish/publish"

# 3. 等用户扫码登录 + 选「图文」+ 上传 5 张图（手动拖 finder）
open ./xhs/

# 4. 标题字段 → 设剪贴板为标题 → Cmd+V
$SKILL/scripts/set-clipboard-text.sh "$(jq -r .title /tmp/xhs.json)"
$SKILL/scripts/send-cmd-v.sh "Google Chrome"

# 5. 正文字段 → 设剪贴板为正文 → Cmd+V
$SKILL/scripts/set-clipboard-text.sh "$(jq -r .body /tmp/xhs.json)"
$SKILL/scripts/send-cmd-v.sh "Google Chrome"

# 6. hashtag 字段 → 手动一个一个 # 加（小红书的 # 是交互输入，不能直接贴）
# jq -r '.hashtags[]' /tmp/xhs.json  # 看清单
```

### 3.2 为什么文本不能像公众号那样一次贴完

小红书创作中心的标题和正文是 **两个独立输入框**，不像公众号那样能一次性贴 HTML。所以脚本必须按顺序：
1. 标题剪贴板 → Cmd+V（用户先点标题框）
2. 正文剪贴板 → Cmd+V（用户点正文框）

### 3.3 hashtag 必须手动加

小红书的 `#话题` 不是普通字符串，而是触发下拉提示后选中的交互式标签。脚本贴进去会变成纯文本，不会被推荐算法识别成话题。

正确做法：
- 在正文最后空一行
- 一个一个手动敲 `#话题名`，等下拉提示出现，选第一个
- 通常 3-5 个就够

## 阶段 4 — 发布前检查（永远人工）

### 4.1 验收清单

- [ ] 标题 ≤ 20 字（含 emoji）
- [ ] 正文 ≤ 1000 字
- [ ] 5 张图全上传成功（顺序：封面 → 痛点 → 流水线 → 数据 → CTA）
- [ ] 封面图比例 3:4
- [ ] hashtag 3-5 个，每个都被识别成话题（蓝色字体）
- [ ] 正文**没有外链 URL**（小红书不允许，会限流）
- [ ] 引导话术不踩雷（避免"私信我" "加我微信"等）

### 4.2 「发布」按钮永远人工

理由（同公众号）：
- 小红书风控对脚本式发布**比微信更严**
- 个人账号被风控后没有申诉渠道
- 发布按钮就在屏幕上，多 0.5 秒人工确认成本可忽略

## 反模式

- ❌ 直接复制公众号原文贴小红书（字数超 4 倍 + 没有 emoji + 语气太严肃 = 必扑街）
- ❌ 文末放 GitHub 链接（外链限流）
- ❌ 16:9 横屏封面（小红书算法不推荐）
- ❌ 全自动登录 + 发布（小红书风控**封号**，不是限流）
- ❌ 让脚本替用户点「发布」按钮
- ❌ 同样的标题 + 图发多个小红书账号（重复内容降权）
- ❌ 把 #话题 写成纯文本（不被算法识别）
- ❌ 标题写"重磅" "揭秘" "震惊"（标题党触发降权）
